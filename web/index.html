Information Retrieval Analysis

Core IR concepts applied to the scraped job corpus: an inverted index that maps tokens to posting lists, TF-IDF ranking to surface discriminative skills, and detailed keyword frequency statistics.

Inverted Index — a data structure that maps each token to the list of documents (job postings) containing it. This is how search engines find relevant documents in milliseconds. The posting list for a token stores every document index where the token appears (with position offsets for phrase queries). The chart and table below show the top tokens by posting-list length (document frequency).

TF-IDF (Term Frequency–Inverse Document Frequency) weights a token by how frequently it appears in a given document, penalised by how common it is across all documents. High TF-IDF → important and discriminative. Low TF-IDF → either ubiquitous (like stopwords) or very rare.
TF-IDF(t, d) = [count(t,d) / |d|] × log( (N+1) / (1 + df(t)) )    |    Score shown = mean TF-IDF across corpus
